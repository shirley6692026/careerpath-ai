- 字体: `-apple-system, 'PingFang SC', 'Microsoft YaHei', sans-serif`

## 二、模块结构

### 2.1 页头区 (Header)
- 左侧:
  - 姓名（22pt 白色 加粗）
  - 求职意向（11pt 浅蓝）
  - 联系方式列表（电话、邮箱、地址等）
- 右侧:
  - 照片区（虚框，可替换为一寸照）
- **无左侧装饰条**

### 2.2 内容模块
每个模块结构：
- 标题栏: `#7EB8F0` 浅蓝底 + `#1A3F6E` 深蓝字
- 内容:
  - 经历条目: 日期左对齐，标题紧随其后
  - 技能: 进度条显示（有AI评分时）或标签显示（无评分时）
  - 评价: 连续文本

## 三、数据流

### 3.1 内容来源
```
原始简历.docx
    ↓ 上传解析
resumeText (原始文本)
    ↓ AI优化
optimizeResult.optimized_text (AI输出文本)
    ↓ mergePersonalInfo()
mergedText (合并个人信息后的完整文本)
    ↓ parseResumeToStructured()
resumeData (结构化数据)
    ↓ generateResumeHTML()
HTML → html2pdf.js → PDF下载
```

### 3.2 个人信息合并逻辑
1. 检查AI输出是否已包含联系方式（`@` 或手机号）
2. 如果缺失，从原始简历前几行提取个人信息：
   - 扫描原始文本前5行
   - 排除模块标题行（教育/经历/技能等）
   - 保留姓名、联系方式、年龄、籍贯等
3. 将个人信息拼接到AI输出文本前面

### 3.3 技能评分数据流
```
AI评分响应 skill_scores
    → optimizeResult.skill_scores
    → resumeData.skillScores
    → PDF技能进度条显示
```

## 四、核心功能

### 4.1 内容保真度
- ✅ "新简历预览" 和 PDF 使用同一份 mergedText 数据源
- ✅ AI优化后必须保留个人信息（姓名/电话/邮箱/年龄/籍贯/政治面貌等）
- ✅ OPTIMIZE_PROMPT 明确要求保留个人信息
- ✅ 前端 mergePersonalInfo() 作为保底补充

### 4.2 技能进度条
- 有AI评分时：`Python ████████░░ 8.0/10`
- 无AI评分时：回退到普通标签 `Python`
- 数据来自 AI 每项技能熟练度评分

### 4.3 预览弹窗
- 去除"简历预览"标题
- 内容区独立滚动
- 按钮区域固定可见
- 一键下载PDF

### 4.4 照片区
- 保留一寸照占位框
- 虚线/实线边框
- 待支持从docx提取头像

## 五、技术栈

| 组件 | 技术 |
|:---|:---|
| PDF生成 | `html2pdf.js`（前端浏览器生成，无后端依赖） |
| HTML模板 | JS模板字符串，`generateResumeHTML(resumeData)` |
| 内容解析 | `parseResumeToStructured(text, targetJob)` |
| 数据传递 | React Context (`AppContext`) |
| AI评分 | 后端 `/api/resume/score` 返回 `skill_scores` |
